# Conventional Commit Message Types

Use these prefixes for commits, pull requests, and code reviews:

- **feat**:     New feature
- **fix**:      Bug fix
- **docs**:     Documentation only changes
- **style**:    Code style (formatting, missing semi colons, etc)
- **refactor**: Code refactoring (no feature or bug fix)
- **test**:     Adding or fixing tests
- **chore**:    Maintenance, build, or auxiliary tools
- **perf**:     Performance improvements
- **ci**:       CI/CD or workflow changes

## Example

``` Text
feat: Add user authentication API
fix: Correct typo in login view
chore: Update dependencies
```

**Follow these conventions for all commits, PRs, and reviews.**
